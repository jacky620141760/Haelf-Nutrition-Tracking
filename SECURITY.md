# Security Policy

## Supported versions

Security fixes are applied to the latest code on the default branch. This project is under active development and does not currently maintain security-supported release branches.

## Reporting a vulnerability

Do not report vulnerabilities, credentials, personal data, or health records in a public issue.

Use GitHub's private security advisory form:

https://github.com/jacky620141760/Haelf-Nutrition-Tracking/security/advisories/new

Include only the information needed to reproduce and assess the issue:

- A description of the vulnerability and its impact
- Affected component, platform, and commit or version
- Minimal reproduction steps or a proof of concept using synthetic data
- Suggested mitigation, if known

Remove access tokens, API keys, account details, health records, photos, and other personal information before submitting a report.

## Response expectations

Maintainers will acknowledge a report when available, investigate it, and coordinate disclosure after a fix or mitigation is ready. Response times are not guaranteed because this is a volunteer-maintained project.

## Scope

Reports about the application code, SQLite data handling, authentication, synchronization, Supabase policies, and Edge Functions are in scope. Vulnerabilities in third-party services should also be reported to the affected service according to its security policy.
